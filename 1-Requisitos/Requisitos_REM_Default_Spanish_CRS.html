    6 Requisitos de información

    IRQ-0001 Órden de compra
    Versión 1.0 ( 26/02/2013 )
    Autores
    • Hans Parra
    • Juan Cuartas
    Fuentes
    • Hans Parra
    • Juan Cuartas
    Dependencias Ninguno
    Descripción El sistema deberá almacenar la información correspondiente a órdenes de compra de acciones. En concreto:
    Datos específicos
    • Id ( Numérico (10). Identificador único de la orden asignado automáticamente por el sistema.)
    • Fecha transaccion ( Date (dd/MM/yyyy HH:mm). Fecha de la transacción colocada automáticamente por el sistema.)
    • Portafolio ( Numérico (10). Id del portafolio en el que se generará la orden de compra. Es una referencia foránea a la tabla de Portafolios.)
    • Fecha inicial ( Date (dd/MM/yyyy HH:mm). Fecha inicial a partir de la cual la orden tendrá validés.)
    • Fecha final ( Date (dd/MM/yyyy HH:mm). Fecha final hasta la cual la orden tendrá validés)
    • Acción ( Numérico (10). Id de la acción comprada. Es una referencia foránea a la tabla de Acciones.)
    • Cantidad ( Numérico (10). Cantidad de acciones ordenadas.)
    • Valor bruto unitario ( Float (20). Valor unitario de cada acción.)
    • Valor neto unitario ( Float (20). Valor unitario real de cada acción. Este valor es calculado automáticamente por el sistema luego de calcular el total de la orden y es igual a Total / Cantidad.)
    • Subtotal ( Float (20). Subtotal calculado automáticamente por el sistema y es igual a Cantidad * Valor bruto unitario.)
    • Porcentaje comisión ( Float (5). Es el porcentaje a pagarle al comisionista sobre el subtotal de la orden.)
    • Valor comisión ( Float (20). Es el valor real de la comisión calculado automáticamente por el sistema y es igual a Subtotal * Porcentaje comisión.)
    • Porcentaje impuestos ( Float (5). Es el porcentaje a pagarle al gobierno sobre el valor de la comisión.)
    • Valor impuestos ( Float (20). Es el valor real de los impuestos calculado automáticamente por el sistema y es igual a Valor comisión * Porcentaje impuestos.)
    • Gastos ( Float (20). Son gastos adicionales que genera la transacción como por ejemplo papelería.)
    • Total ( Float (20). Es el valor total de la orden que debe pagar el inversionista y es igual a Subtotal + Valor comisión + Valor impuestos + Valor gastos)
    • Observaciones ( String (500). Observaciones adicionales ingresadas por el inversionista.)
    • Estado ( Numérico (1). Estado actual de la orden, el cual es administrado internamente por el sistema. Puede tomar los siguientes valores: 1 = PENDIENTE, 2 = EJECUTADA y 3 = CANCELADA.)
    • Inversionista ( Numérico (10). Id del inversionista que genero la orden. Es una referencia foránea a la tabla de Inversionistas.)
    • Trader ( Numérico (10). Id del trader asignado automáticamente por el sistema para procesar la transacción. Es una referencia foránea a la tabla de Traders.)
    Tiempo de vida Medio Máximo
    10 año(s) 10 año(s)
    Ocurrencias simultáneas Medio Máximo

    UC-0013 Ingresar al sistema
    Versión 1.0 ( 05/11/2012 )
    Autores
    • Hans Parra
    • Juan Cuartas
    Fuentes
    • Hans Parra
    • Juan Cuartas
    Dependencias Ninguno
    Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando un usuario accede a la aplicación a través del formulario de login. Este escenario aplica tanto para el Sistema Web, como para el Sistema Móvil.
    Precondición El Login debe existir, la Clave debe ser correcta y el usuario se debe encontrar en estado Activo
    Secuencia normal Paso Acción
    1 El actor Usuario (ACT-0004) ingresa la URL principal de la aplicación a través de su navegador Web
    2 El sistema muestra el formulario de inicio, presentando el campo Login, Clave y el botón "Ingresar"
    3 El actor Usuario (ACT-0004) digita su Login y su Clave y hace click en el boton "Ingresar"
    4 El sistema valida que el Login exista, que la Clave sea correcta y que el usuario se encuentre Activo
    5 El sistema almacena en el log de accesos la siguiente informacion: Usuario, Fecha, Hora, Ip, Navegador y Sistema Operativo de la petición
    6 El sistema almacena las credenciales del usuario en la sesión actual
    7 El sistema hace una redirección a la página principal del aplicativo, mostrando las opciones a las que tiene acceso el usuario de acuerdo a su Perfil
    Postcondición El sistema creará un log de acceso con los datos de la petición y almacenará las credenciales del usuario logueado en la sesión actual
    Excepciones Paso Acción
    4 Si el Login no existe, el sistema muestra mensaje: "El Login no existe" y vuelve al paso 1, a continuación este caso de uso continúa
    4 Si la Clave es incorrecta, el sistema muestra mensaje: "La Clave es incorrecta" y vuelve al paso 1, a continuación este caso de uso continúa
    4 Si el usuario se encuentra Inactivo, el sistema muestra mensaje: "Usted se encuentra inactivo en nuestro Sistema", a continuación este caso de uso queda sin efecto
    Rendimiento Paso Tiempo máximo

    UC-0004 Crear portafolio
    Versión 1.0 ( 13/09/2012 )
    Autores
    • Hans Parra
    • Juan Cuartas
    Fuentes
    • Hans Parra
    • Juan Cuartas
    Dependencias Ninguno
    Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando un inversionista desea crear un nuevo portafolio de inversión desde su dispositivo móvil.
    Precondición El nombre del portafolio debe ser único para el inversionista
    Secuencia normal Paso Acción
    1 El actor Inversionista (ACT-0002) hace click en el botón "Crear Portafolio" desde la página principal del aplicativo móvil
    2 El sistema muestra el formulario para la creación del portafolio, presentando la lista de comisionistas y de divisas matriculados desde el Sistema Web
    3 El actor Inversionista (ACT-0002) ingresa el nombre del portafolio
    4 El actor Inversionista (ACT-0002) selecciona el comisionista que se encargará de administrar el portafolio
    5 El actor Inversionista (ACT-0002) selecciona la divisa con la cual se calculará el saldo y el balance del portafolio
    6 El actor Inversionista (ACT-0002) hace click en el boton "Guardar"
    7 El sistema valida que el nombre del portafolio sea único
    8 El sistema almacena el nuevo portafolio en la base de datos
    9 El sistema muestra mensaje "El portafolio se registró correctamente"
    Postcondición Se creará un nuevo portafolio con Saldo en 0, Fecha de Apertura = Fecha Actual y Estado = Activo
    Excepciones Paso Acción
    7 Si el nombre del portafolio ya se encuentra registrado para el inversionista, el sistema muestra mensaje: "Ya existe un portafolio con este nombre" y vuelve al paso 3, a continuación este caso de uso continúa

    UC-0011 Generar balance
    Versión 1.0 ( 13/09/2012 )
    Autores
    • Hans Parra
    • Juan Cuartas
    Fuentes
    • Hans Parra
    • Juan Cuartas
    Dependencias Ninguno
    Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando un inversionista desea consultar el balance general de un portafolio en específico desde su dispositivo móvil.
    Precondición El inversionista debe seleccionar un portafolio al cual generar su balance
    Secuencia normal Paso Acción
    1 El sistema despliega la lista de portafolios matriculados para el inversionista desde la página principal del aplicativo móvil
    2 El actor Inversionista (ACT-0002) selecciona un portafolio y a continuación hace click en el botón "Balance"
    3 El sistema valida que el inversionista haya seleccionado un portafolio al cual generar su balance
    4 El sistema muestra la información básica del portafolio: Nombre, Fecha de Apertura y Saldo
    5 El sistema calcula la Utilidad del portafolio (Ingresos - Egresos) y la presenta al usuario
    6 El sistema calcula la Rentabilidad del portafolio ( Utilidad / Días) y la presenta al usuario
    7 El sistema muestra el inventario de acciones pertenecientes al portafolio, que se presenta como una tabla con las siguientes columnas: Nombre de la Acción y Cantidad
    Postcondición El sistema mostrará el balance general del portafolio seleccionado
    Excepciones Paso Acción
    3 Si el inversionista no seleccionó ningún portafolio, el sistema muestra mensaje: "Seleccione un Portafolio" y vuelve al paso 2, a continuación este caso de uso continúa

    UC-0007 Colocar orden de compra
    Versión 1.0 ( 13/09/2012 )
    Autores
    • Hans Parra
    • Juan Cuartas
    Fuentes
    • Hans Parra
    • Juan Cuartas
    Dependencias Ninguno
    Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando un inversionista desea colocar una orden de compra de acciones desde su dispositivo móvil.
    Precondición Se tiene el saldo suficiente para respaldar la operación, las fechas son válidas, la cantidad de acciones está en el rango permitido, el valor unitario está en el rango permitido, existe un trader al cual asignarle la orden
    Secuencia normal Paso Acción
    1 El sistema despliega la lista de portafolios matriculados para el inversionista desde la página principal del aplicativo móvil
    2 El actor Inversionista (ACT-0002) selecciona un portafolio y a continuación hace click en el botón "Agregar Orden de Compra"
    3 El sistema valida que el inversionista haya seleccionado un portafolio al cual generar la orden de compra
    4 El sistema despliega el formulario para la adición de la orden de compra con todos sus campos
    5 El actor Inversionista (ACT-0002) ingresa la fecha inicial y final en la que la orden tendrá validez
    6 El sistema valida que la fecha final sea mayor o igual a la fecha inicial, y que la fecha inicial sea mayor o igual a la fecha actual
    7 El actor Inversionista (ACT-0002) selecciona la accion que desea ordenar

    10 Glosario

    Acción:Una acción es la parte alícuota del capital social de una sociedad anónima. Cabe resaltar que poseer acciones de una compañía confiere legitimidad al accionista para exigir sus derechos, pero a su vez para cumplir con sus obligaciones.

    BVC:Abreviatura de la Bolsa de Valores de Colombia. Es el único mercado de acciones y otros valores de Colombia, organizado a través de la estructura de bolsa.

    Capital:En el contexto de nuestro sistema, el capital corresponde al saldo en efectivo que tiene un portafolio, en favor del inversionista.

    Comisionista:Son los únicos entes autorizados por la BVC para vender o comprar en el mercado accionario. Cuando un inversionista realiza una compra, realmente es la firma comisionista la que hace la compra y luego le sede estos activos a través de una transacción.

    Deposito:Es una operación en la que un inversionista le hace un ingreso de efectivo a un portafolio.

    Dividendo:El dividendo activo, es la parte del beneficio obtenido por las sociedades mercantiles cuyos órganos sociales acuerdan que sea repartido entre los socios de las mismas. Es decir, una vez acordado su reparto, es un crédito del socio frente a la sociedad. De otro lado, el dividendo pasivo es el crédito que ostenta la sociedad mercantil frente al socio, por la parte del capital social que suscribió y que se comprometió a desembolsar. La diferencia entre las aportaciones y el desembolso inicial de los accionistas.

    Enrepar Acciones:Es la acción de realizar una operación Repo por parte de un inversionista: Esta operación es como un préstamo, en donde un inversionista le pide dinero prestado a una firma comisionista y esta le concede el préstamo respaldado con las acciones que el inversionista tenga en su portafolio.

    Estado financiero:Es el saldo actual en capital que tiene un inversionista en un portafolio y la cantidad de acciones que posee, los cuales son reconocidos por la firma comisionista que administra el portafolio.

    Inversionista:Es aquella persona natural (persona física) o jurídica que es propietaria de acciones de los distintos tipos de sociedades anónimas o comanditarias que pueden existir en el marco jurídico de cada país. En el caso de nuestro sistema, es el usuario que compra o vende acciones.

    Mercado Financiero:Es un espacio en el que se realizan los intercambios de instrumentos financieros y se definen sus precios.

    Nemotecnico:Es un sistema sencillo utilizado para recordar una secuencia de datos, nombres, números, y en general para recordar listas de items que no pueden recordarse fácilmente.

    Orden de compra:Es una operación en la que un inversionista le ordena a un trader la compra de una cantidad determinada de acciones a un cierto valor y dentro de un determinado rango de fechas establecido por el inversionista.

    Orden de Venta:Es una operación en la que un inversionista le ordena a un trader la venta de una cantidad determinada de acciones a un cierto valor y dentro de un determinado rango de fechas establecido por el inversionista.

    Portafolio:La cartera de inversiones o portafolio de inversiones, es el conjunto de activos financieros en los cuales se invierte, para el caso de nuestro sistema, estos activos pueden ser acciones o capital.

    Repo:Es una operación con pacto de recompra, es decir, una entidad financiera vende un activo con un pacto de recompra por un precio determinado dentro de un tiempo determinado. Esta operación también se conoce como reporto, Repurchase Agreement o Sale and Repurchase Agreement.

    Retiro:Es una operación en la que un inversionista le hace un retiro de efectivo a un portafolio.

    Trader:Persona que realiza la compra, venta de acciones en la Bolsa de Valores en nombre del inversionista que pone la orden.
